# Number Processor

This project, Number Processor, is designed to generate a sequence of numbers and then double each number in the sequence. It is built using the Aineko framework and consists of two primary nodes within the pipeline.

## Nodes

### NumberGeneratorNode

The `NumberGeneratorNode` is responsible for generating a sequence of numbers. It starts from a defined beginning number and continues to emit numbers in the sequence until it reaches the defined end. The start and end values can be configured in the pipeline configuration file.

### NumberDoublerNode

The `NumberDoublerNode` consumes the sequence of numbers generated by the `NumberGeneratorNode` and doubles each number. The result is then produced to another dataset for further consumption or storage.

## Pipeline Design

The pipeline is configured in such a way that the `NumberGeneratorNode` produces a stream of numbers to the `number_sequence` dataset. The `NumberDoublerNode` is set up to consume from this dataset and produce the doubled numbers to the `doubled_numbers` dataset.

The pipeline is designed to be efficient and scalable, utilizing the Aineko framework's ability to handle streaming data and perform real-time processing. The nodes are stateless, making them easy to scale horizontally if needed.

## Usage

To run this pipeline, ensure that you have the Aineko framework installed and configured correctly. Update the `pipeline.yml` file with the desired start and end values for the number sequence. Then, execute the pipeline using the Aineko CLI tools.

The `NumberGeneratorNode` will begin emitting numbers, which will be immediately doubled by the `NumberDoublerNode`, and the results will be available in the `doubled_numbers` dataset.

This project showcases the simplicity and power of the Aineko framework for building data processing pipelines. It is a basic example that can be extended for more complex processing tasks.
